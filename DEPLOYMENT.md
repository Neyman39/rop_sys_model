# Деплой на GitHub Pages

## Автоматический деплой

Проект настроен для автоматического деплоя на GitHub Pages при каждом push в ветку `main`.

### Настройка репозитория

1. **Создайте репозиторий на GitHub** (если еще не создан)

2. **Включите GitHub Pages в настройках репозитория:**
   - Перейдите в Settings → Pages
   - В разделе "Source" выберите "GitHub Actions"

3. **Отправьте код в репозиторий:**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ваш-username/название-репозитория.git
   git push -u origin main
   ```

4. **Дождитесь завершения деплоя:**
   - Перейдите во вкладку "Actions" в репозитории
   - Дождитесь завершения workflow "Deploy to GitHub Pages"
   - После успешного деплоя сайт будет доступен по адресу: `https://ваш-username.github.io/название-репозитория/`

## Важно

Проект использует **HashRouter** для маршрутизации, что обеспечивает корректную работу на GitHub Pages без необходимости дополнительной настройки сервера. URL будут выглядеть так: `https://ваш-username.github.io/название-репозитория/#/`

## Ручной деплой

Если вы хотите развернуть сайт вручную:

1. **Соберите проект:**
   ```bash
   pnpm install
   pnpm run build
   ```

2. **Разверните содержимое папки `dist` на любом хостинге**

## Локальная разработка

Для запуска проекта локально:

```bash
# Установка зависимостей
pnpm install

# Запуск dev сервера
pnpm run dev

# Сборка для продакшена
pnpm run build

# Предварительный просмотр production сборки
pnpm run preview
```

Сайт будет доступен по адресу: `http://localhost:5173`

## Особенности

- Страница со списком контрагентов доступна без авторизации
- Для просмотра детальной информации о контрагенте требуется авторизация
- Раздел администрирования доступен только для пользователей с ролью "Администратор"
- Красные акценты используются для активных разделов навигации и кнопки входа

## Структура проекта

- `/src/app/` - основные компоненты приложения
- `/src/app/pages/` - страницы приложения
- `/src/app/components/` - переиспользуемые компоненты
- `/src/app/contexts/` - React контексты (AuthContext)
- `/src/app/data/` - mock данные
- `/src/styles/` - стили приложения
